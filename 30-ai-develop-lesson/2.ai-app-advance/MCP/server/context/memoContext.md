#要求
总结对话记录提取出用户的一些特点

#生成具体要求
按下面json生成，并且返回一个json，纯粹的返回json就好，不需要携带任何的md格式。仔细阅读json数据的属性的含义，如果没有提取到该属性的信息，则返回空字符串，不生成没提到的属性
sf-用户的身份特点,根据sf对象里的属性，进一步生成身份的具体点
like-用户的喜好特点,根据like对象里的属性，进一步生成喜好的具体点
status-用户最近的情感和身体状态，根据status对象里的属性，进一步生成最近状态的具体点
```json
{
  "sf":{
    "career":"职业",
    "location":"居住地"
  },
  "like":{
    "food":"喜欢的食物，值是个数组，里面存放用户喜好的食物种类，没提取到则为空数组",
    "sport":"喜欢的运动，值是个数组，里面存放用户喜好的食物种类，没提取到则为空数组",
  },
  "stauts":{
     "feel":"最近的情感状态，字符串一句话总结",
     "body":"最近的身体状态，字符串一句话总结"
  }
}
```
