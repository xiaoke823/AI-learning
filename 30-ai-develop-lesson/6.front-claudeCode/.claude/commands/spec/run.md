## 重要前提

以此命令去开发需求时，不要直接产出代码，一定要按步骤去产出 md 格式的文档，文档备齐后，根据文档再进行代码生成。文档产出后需要用户修改确认，确认后在进行下一步。

## 文档保存位置

产出的文档保存在当前项目的 spec 文件夹内，如果不存在 spec 文件夹则创建。如果文档已经存在得覆盖原文档

## 步骤

1. 产出需求文档，根据 user 提问的需求，生成需求文档，主要说明界面上的操作动线。文档生成后，在写入前给用户修改确认。确认后以`design.md`为名字写入到 spec 文件夹
2. 产出需求文档后，根据需求文档和 user 的需求产出技术文档。说明代码里的代码逻辑步骤，以及会用到哪些库。文档生成后，在写入前给用户修改确认。确认后以`technology.md`为名字写入到 spec 文件夹
3. 前面两步完成后，根据之前产出的文档结合用户的需求，生成代码，代码要分模块，按照用户的目录情况写入。生成代码要结合用户当前的代码情况，尽量少修改，简单实现。
