当你生成记忆相关文件请仔细遵循以下内容

## 目标
生成项目级和用户级两个记忆，用memory工具写入。

## 记忆点
1. 项目特点-项目的结构，技术栈，主要功能的总结（项目级记忆）
2. 项目功能开发记录，按照功能描述，主要涉及文件记录（项目级记忆）
3. 项目常见错误，用户纠正的问题和提出的bug（项目级记忆）
4. 用户的开发偏好，JavaScript风格偏好，css风格偏好，实现功能时的侧重点(用户级)
5. 用户的技术情况，用什么语言，用什么技术栈(用户级)
6. 用户项目情况，主要说明用户有哪些项目。每个项目说明项目的功能，目录地址，技术栈。不要顶掉之前的项目，除非用户明确让你删了（用户级）
** 重要提示 **
项目级别记忆写入到项目记忆路径
用户级记忆写入到用户记忆路径


## 怎么记录
基于目前现有的记忆资料。结合本次拿到的对话记录，以及提供的上下文。综合修改记忆资料。
### 当前项目记忆:
文件内容:
``` markdowm
  ${projectMeory}
```
### 当前user全局记忆
``` markdowm
  ${userMeory}
```
### 当前项目上下文:
文件内容:
``` markdowm
  ${projectMd}
```
### 用户全局上下文:
文件内容:
``` markdowm
  ${userMd}
```
### 要记忆的对话记录
内容：${record}


### 记忆文件目录
结合以上资料，按照记忆点去提取记录中的重点。提取出相关需要记忆的内容。
请记住记忆点分为项目级和用户级，他们存放的位置不同：
项目级的记忆，生成在项目目录下的.front/memorys里，文件名叫在memorys.md
用户级的记忆，生成在user目录下的.front/memorys里，文件名叫在memorys.md

## 记忆规范
1. 如果上下文里已经存在的内容，则不再生成到记忆文件里
2. 一定要基于已有文件，结合新给入的上下文和对话记录。对记忆文件更新迭代，不要彻底颠覆之前的记忆
3. 如果记忆文件中提到的内容和新给入的上下文和对话记录有不可兼容的冲突，以新给入内容为准。
4. 和技术无关的内容，请忽略，不要生成任何记忆
5. 谨慎记忆，如果目前内容和记忆文件无太大差别，则保留原状
6. 记忆用户的用户项目情况，除非用户明确要求，否则不要去掉之前的记录，叠加记录新的项目情况
