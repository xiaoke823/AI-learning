# 需求文档 · `/` 指令选择 与 `@` 文件选择

> 本文档描述最终实现的界面操作动线，与代码保持一致。

## 一、功能概述

在现有终端 AI 助手基础上，新增两个"内嵌式"快捷交互：

1. **`/` 指令菜单**：输入 `/` 时弹出可用指令列表，支持上下选择、Tab 确认、边输入边筛选。
2. **`@` 文件菜单**：输入 `@` 时弹出当前项目的文件列表，选中后把该文件内容作为上下文，随提问一起发给大模型。

两者都不改变"回车发送对话"的整体动线，只是在用户输入过程中提供"浮层式"的选择辅助。

---

## 二、`/` 指令选择

### 2.1 触发条件
- 用户在输入行按下 `/`，且该 `/` 处于"命令起始位置"时触发菜单：
  - 即 `/` 位于**行首**，或**前面紧跟一个空格**；
- 触发后，输入框保留已输入的 `/`（及后续字符），下方弹出指令列表浮层。

### 2.2 指令列表来源
- 来自项目内置命令（可后续扩展）：
  - `/help` —— 显示帮助信息
  - `/clear` —— 清空屏幕上的对话显示（保留对话历史上下文）
  - `/exit`、`/quit` —— 退出程序

### 2.3 列表展示
- 在当前输入行**下方**渲染一个列表，每行一条指令，格式示例：
  ```
  /help    显示帮助信息
  /clear   清空屏幕显示(保留历史)
  /exit    退出程序
  ```
- 当前选中项**整行高亮**（青色 + `▶` 前缀），与 `@` 菜单风格一致。
- 列表最多展示固定行数（指令菜单 6 条），超出部分靠上下键滚动。

### 2.4 键盘操作动线
| 按键 | 行为 |
| --- | --- |
| 继续输入字符（`a-z` 等） | 作为筛选关键词，**实时**过滤匹配的指令（命令名或描述中包含已输入内容） |
| `↑` / `↓` | 在（筛选后的）列表中上/下移动选中项 |
| `Tab` 或 `Enter` | 确认选择：把选中指令**完整命令名**填入输入行（替换 `/` 后的草稿），**列表立即消失**，光标停在该指令后，**等待用户回车发送** |
| `Esc` | 取消菜单，回到普通输入状态，保留已输入内容 |
| 空格 | 关闭菜单（表示已完成命令输入） |
| 退格删掉 `/` | 菜单自动关闭 |

### 2.5 筛选规则
- 用 `/` 之后输入的字符片段，在指令列表中做**包含匹配**（不区分大小写）；
- 无匹配项时，列表区域显示提示（如"（无匹配项）"），用户仍可继续输入或回退。

### 2.6 选择与执行的关系
- **选择指令 ≠ 执行指令**：
  - 用户用 `Tab`/`Enter` 选定指令后，列表消失，输入行变成完整的 `/xxx`（如 `/help`），但**此时不执行任何动作**；
  - 用户可继续编辑，或直接**回车确认发送**；
  - **只有回车发送后**，才由命令处理逻辑（`runCommand`）解析并真正执行该指令。
- 指令的执行时机与普通对话一致——统一通过"回车发送"触发。

## 三、`@` 文件选择

### 3.1 触发条件
- 用户在输入行任意位置输入 `@` 时，弹出当前项目的文件列表浮层。
- （与 `/` 不同，`@` 可以出现在一句话中间，例如 `帮我看看 @app.js 的逻辑`。）

### 3.2 文件列表来源
- 扫描**当前终端工作目录**（`process.cwd()`）下的所有文件，**递归**读取。
- **排除目录**：`node_modules`、`.git`、`.front-claude`、`dist`、`build`。
- 文件以**相对路径**展示（相对当前工作目录）。

### 3.3 列表展示
- 在输入行下方渲染文件列表，每行一个相对路径；
- 选中项**整行高亮**（青色 + `▶` 前缀）；
- 文件较多时，配合输入筛选展示前若干条（最多 8 条），超出靠上下键滚动。

### 3.4 键盘操作动线
| 按键 | 行为 |
| --- | --- |
| 继续输入字符 | 作为筛选关键词，对文件**路径/文件名**做包含匹配，实时过滤 |
| `↑` / `↓` | 上/下移动选中项 |
| `Tab` 或 `Enter` | 确认：将选中文件以 `@相对路径` 形式插入输入行（替换 `@` 后的草稿），菜单关闭，并**记录该文件待作为上下文** |
| `Esc` | 取消菜单，回到普通输入 |
| 退格删掉 `@` | 菜单自动关闭 |

### 3.5 确认后（携带上下文）
- 输入行中插入 `@相对路径` 标记；
- 用户可继续输入提问内容，也可再输入 `@` 选择更多文件（支持多选）。
- 当用户**回车发送**这条消息时：
  - 读取所有 `@` 标记对应文件的**内容**；
  - 把文件内容与用户原始提问**拼装**后，作为一条 `user` 消息发给大模型。

### 3.6 发送给大模型的消息格式
- 把附加文件内容以结构化文本拼到提问前，例如：
  ```
  【参考资料：以下为用户通过 @ 附加的文件】
  --- 文件：app.js ---
  <文件内容>
  --- 文件结束 ---

  用户问题：<用户输入的提问>
  ```

## 四、整体动线小结

```
启动 → 显示欢迎 → 进入输入框（自定义输入控件）
   │
   ├─ 输入普通文本 ────────────────→ 回车 → 保留输入行 + 发给大模型
   ├─ 输入 `/`（命令起始位置） ────→ 弹出指令菜单 → 筛选/上下/Tab确认（列表消失）→ 回车发送 → 解析执行
   └─ 输入 `@` ─────────────────→ 弹出文件菜单 → 筛选/上下/Tab确认 → 标记附加文件
                                                                  │
                                                          回车发送时携带文件内容
```

- 三种状态（普通输入 / 指令菜单 / 文件菜单）互斥，同一时刻只处于一种；
- 任一菜单都可通过 `Esc` 或退格删掉触发符退出，回到普通输入；
- 回车发送后，用户输入行会保留在屏幕上方，AI 回复显示在其下方，最后出现新的输入提示符。

---

## 五、边界与约定

1. `/` 菜单仅在"命令起始位置"触发，避免普通文本里的 `/`（路径、日期）误触；`@` 菜单在任意位置均可触发。
2. 文件扫描需排除依赖与构建目录；若文件极多，靠输入筛选 + 列表上限控制渲染量。
3. 附加文件读取失败（不存在、无权限）时，提示用户并跳过该文件，不阻断整次发送。
4. `/clear` 仅清空屏幕显示，**不清空对话历史**（`messages` 保留，大模型仍记得之前的内容）。
5. 指令的执行统一由"回车发送"触发；菜单选择只负责填入命令名，不触发执行。
